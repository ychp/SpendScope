# SpendScope 额度与周期指标强化设计

日期：2026-07-10

## 目标

在现有 `920 × 620` 无滚动详细看板中，提高顶部复合卡的信息密度与可读性：强化四个周期的标题和 Token 总量，并将额度百分比与重置时间重新组织为更直观的双环结构。

## 布局结构

- 详细看板窗口尺寸保持 `920 × 620`，不增加滚动容器。
- 顶部复合卡高度由 190 点提高到约 250–260 点。
- 左侧额度区域约占复合卡内部宽度的 35%，右侧周期区域约占 65%。
- 左侧与右侧之间保留垂直分隔线。
- 下方 Token 趋势与 Token 构成区域继续完整显示。

## 左侧额度区域

### 双环

- 外环表示 5 小时额度，继续使用紫色。
- 内环表示 7 天额度，继续使用蓝色。
- 双环整体在左侧区域内水平居中，并为下方重置时间预留空间。

### 圆心信息

- 移除“当前额度”和套餐名称。
- 圆心以两行展示剩余额度：
  - 第一行：`5H 85%`；
  - 第二行：`7d 84%`。
- 两个周期必须换行展示，百分比使用较强字重，周期缩写略小。

### 重置时间

- 不再在圆环右侧显示额度图例。
- 两个重置时间放在圆环正下方，纵向排列，并使用与环一致的颜色圆点：
  - `5H  02:52`；
  - `7d  2026-07-13 10:45`。
- 不展示“重置”二字。
- 日期统一使用 `YYYY-MM-DD HH:mm` 格式。

## 右侧周期四宫格

- 继续使用 2×2 四宫格，顺序保持为今日、7 日、30 日、累计。
- 每个宫格的周期标题使用约 14 点半粗体。
- Token 总量使用约 24 点粗体，并保持等宽数字。
- 每个宫格内的四个分项使用 2×2 两行布局：
  - 第一行：输入、缓存；
  - 第二行：输出、推理。
- 输入表示未缓存输入；输出表示不包含推理的可见输出。
- 四个分项之和必须等于该周期 Token 总量。

## 视觉约束

- 延续 SwiftUI 原生浅色/深色自适应语义颜色。
- 保留轻量卡片背景、圆角和低对比度边框。
- 不新增图表、进度条、控制项或滚动条。
- 菜单栏弹窗及其信息结构不在本次修改范围内。

## 验收标准

- 顶部复合卡高度明显高于当前 190 点版本，且窗口仍无滚动条。
- 双环圆心严格分两行显示 `5H 85%` 与 `7d 84%`。
- 圆环右侧不再出现百分比、周期说明或重置时间。
- 圆环下方显示两行重置时间，且日期格式为 `YYYY-MM-DD HH:mm`。
- 四个周期标题和 Token 总量比当前版本更醒目。
- 每个周期的输入、缓存、输出、推理采用 2×2 两行布局。
- `920 × 620` 最小窗口中，顶部与下方区域均完整可见。
- Xcode Debug 构建和现有单元测试通过。
