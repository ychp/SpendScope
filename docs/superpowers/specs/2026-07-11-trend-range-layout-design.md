# SpendScope 趋势范围与底部布局设计

日期：2026-07-11

## 目标

简化详细看板底部区域：移除 Token 构成卡，将 Token 趋势扩展为全宽，并把时间范围选择器从页面顶部移动到趋势卡标题栏。时间范围只影响趋势图，不影响顶部额度和四周期统计。

## 页面布局

- 页面顶部标题区只保留：
  - `SpendScope` 标题；
  - Codex 套餐名称；
  - 刷新状态。
- 从页面顶部移除时间范围选择器。
- 顶部额度与四周期复合卡保持现有布局和数据，不受时间范围影响。
- 删除右下方 Token 构成卡及其展示辅助结构。
- 底部只保留一张 Token 趋势卡，并占满可用宽度和高度。

## 趋势卡标题栏

- 左侧显示“Token 趋势”。
- 右侧显示分段选择器：`今日`、`7 天`、`30 天`、`全部`。
- 分段选择器宽度保持约 260 点。
- 默认选择 `7 天`。
- 标题与选择器位于同一水平行，不额外占用图表行。

## 时间范围行为

- 使用类型安全的趋势范围枚举管理选择状态，避免散落字符串判断。
- 选择结果只过滤 `dailyUsage`：
  - 今日：取按时间排序后的最后 1 条日用量；
  - 7 天：取最后 7 条；
  - 30 天：取最后 30 条；
  - 全部：返回全部日用量。
- 当可用数据少于目标天数时，展示全部可用数据，不报错、不补零。
- 输入数据为空时，趋势图保持空状态且应用不崩溃。
- 过滤不修改原始快照，也不影响额度、今日、7 日、30 日和累计四周期指标。

## 趋势数据与图表

- 预览数据扩展到至少 45 个按时间升序排列的日用量点，使四个范围在开发版本中可被验证。
- 趋势图继续使用现有面积图、折线和节点样式。
- 横轴继续展示日期标签；在 30 天和全部范围下由 Charts 自动控制标签密度，避免全部日期同时显示。
- 趋势图使用趋势卡剩余的全部可用高度，不保留原固定 180 点高度限制。

## 移除内容

- 删除 `compositionCard`。
- 删除 `breakdownItems` 和仅供该卡片使用的 `BreakdownDisplayItem`。
- 保留领域模型中的 `TokenBreakdown` 与 `DashboardSnapshot.breakdown`，因为它们仍可供菜单栏或后续功能使用；本次不做无关模型重构。

## 测试与验收

- 为趋势范围过滤增加单元测试，覆盖 1、7、30、全部、数据不足和空数组。
- 默认范围为 7 天。
- 切换范围时，趋势图数据点数量与范围一致。
- 顶部额度与四周期指标在切换范围时保持不变。
- 详细窗口不再显示 Token 构成卡。
- 趋势卡横向占满底部区域，时间选择器位于卡片标题栏右侧。
- `920 × 620` 窗口内无滚动条和内容裁切。
- Xcode Debug 构建与完整 XCTest 通过。
