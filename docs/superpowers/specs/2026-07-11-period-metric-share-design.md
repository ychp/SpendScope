# SpendScope 周期分项占比设计

日期：2026-07-11

## 目标

优化详细看板右侧四个周期卡片的分项展示，使用户能同时读取输入、缓存、输出、推理的 Token 用量，以及每个分项占当前周期总量的比例。

## 适用范围

- 应用于今日、7 日、30 日、累计四个周期卡片。
- 保持顶部复合卡、左侧额度双环、窗口尺寸和下方图表不变。
- 不修改菜单栏弹窗。

## 分项布局

- 每个周期卡片继续使用 2×2 分项网格：
  - 第一行：输入、缓存；
  - 第二行：输出、推理。
- 每个分项内部按三层排列：
  1. 彩色圆点和分项名称；
  2. 左侧 Token 用量、右侧占比；
  3. 细圆角进度条。
- 输入使用紫色，缓存使用蓝色，输出使用橙色，推理使用青色。

## 数值口径

- 输入 = 未缓存输入。
- 缓存 = 缓存输入。
- 输出 = 不包含推理的可见输出。
- 推理 = 推理输出。
- 分项占比计算公式：`分项 Token ÷ 当前周期 Token 总量 × 100%`。
- 占比显示一位小数，例如 `48.2%`。
- Token 用量继续使用现有紧凑格式，例如 `8.2M`、`700.0K`。
- 当周期总量为 0 时，占比显示 `0.0%`，进度条为空，避免除零。
- 计算结果限制在 `0...1`，避免异常数据导致进度条溢出。

## 视觉层级

- 周期名称和周期总量保持当前醒目字号。
- 分项 Token 用量使用半粗体，作为分项的主要信息。
- 占比使用较弱的次级文本颜色，避免与用量争夺视觉焦点。
- 进度条高度约 3–4 点，背景使用低透明度中性色，填充色与分项圆点一致。
- 进度条不显示系统标签，不引入额外边框或阴影。

## 布局约束

- 顶部复合卡高度保持 260 点。
- 每个周期卡片必须在现有四宫格中完整显示，不增加滚动条。
- 分项内部使用紧凑间距，确保 2×2 网格在 `920 × 620` 最小窗口中不裁切。
- 长数值允许按现有规则缩放，但占比文本不得换行。

## 测试与验收

- 为占比计算增加单元测试，覆盖正常值、总量为零和异常超界值。
- 今日示例中应显示：
  - 输入 `8.2M`、`48.2%`；
  - 缓存 `7.9M`、`46.5%`；
  - 输出 `700.0K`、`4.1%`；
  - 推理 `200.0K`、`1.2%`。
- 四个周期均展示用量、占比和对应颜色的细进度条。
- 详细窗口不出现滚动条，左侧额度区域与下方图表不受影响。
- Xcode Debug 构建和完整 XCTest 通过。
