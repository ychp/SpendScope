# SpendScope 紧凑看板设计

日期：2026-07-10

## 目标

将详细看板从带滚动条的 `1040 × 700` 布局调整为 `920 × 620` 的单页紧凑布局，并将顶部摘要与额度区域合并为一张复合卡，在一个窗口内完整展示核心信息。

## 设计原则

- 将原顶部摘要行和额度/模型行合并，减少纵向占用。
- 完全移除 `ScrollView`，正常窗口尺寸下不出现滚动条。
- 默认窗口尺寸和最小窗口尺寸均为 `920 × 620`。
- 允许用户放大窗口，但不允许缩小到内容发生裁切的尺寸。
- 保持菜单栏弹窗不变。
- 保留下方 Token 构成卡片；从顶部区域移除模型分布卡片。

## 布局结构

1. 顶部：产品名称、套餐与刷新状态、时间范围选择器。
2. 中部：一张横向复合卡，左侧展示当前额度，右侧以四宫格展示今日、7 日、30 日和累计 Token 指标。
3. 底部：Token 趋势与 Token 构成左右排列。

## 顶部复合卡

### 左侧：当前额度

- 左侧固定宽度为 330 点，约占复合卡宽度的 38%。
- 使用同心双环展示两档剩余额度。
- 外环表示 5 小时额度，使用紫色。
- 内环表示 7 天额度，使用蓝色。
- 双环中心显示当前套餐名称。
- 双环右侧使用与环同色的圆点作为图例，分别展示：
  - 窗口名称；
  - 剩余百分比；
  - 重置时间。

### 右侧：周期指标四宫格

- 右侧使用 2×2 四宫格，每个宫格具有独立的轻量背景与边界。
- 四个周期的位置固定为：
  - 左上：今日；
  - 右上：7 日；
  - 左下：30 日；
  - 右下：累计。
- 每个宫格显示周期名称、Token 总量，以及四个明细维度：
  - 输入，表示未缓存输入；
  - 缓存；
  - 输出，表示不包含推理的可见输出；
  - 推理。
- 不展示额外的“总额度使用进度条”。
- 不在该区域展示模型分布。

### Token 口径

- 输入 = 未缓存输入 = `input_tokens - cached_input_tokens`。
- 缓存输入 = `cached_input_tokens`。
- 输出 = 可见输出 = `output_tokens - reasoning_output_tokens`。
- 推理 = `reasoning_output_tokens`。
- 周期总量 = 输入 + 缓存 + 输出 + 推理。
- 顶部四宫格与下方 Token 构成卡片使用完全相同的四类口径，避免重复计数。

## 尺寸调整

- 窗口：`1040 × 700` 调整为 `920 × 620`。
- 页面外边距：24 点调整为 16 点。
- 页面主间距：18 点调整为 10 点。
- 卡片默认内边距：18 点调整为 12 点。
- 周期总量字号：30 点调整为 22 点。
- 外层额度环直径约 138 点，内层额度环直径约 92 点。
- 顶部复合卡高度约 190 点。
- 趋势图高度：260 点调整为 180 点。
- Token 构成区域宽度：360 点调整为 300 点。
- 时间范围选择器宽度：300 点调整为 260 点。

## 自适应行为

- 根布局使用普通 `VStack`，不使用滚动容器。
- 复合卡左侧固定 330 点，右侧四宫格使用全部剩余空间。
- 四宫格使用两列两行等宽等高布局，宫格间距约 8 点。
- 底部趋势与构成卡片使用相同高度约束。
- 窗口放大时，周期指标和趋势图可获得额外横向空间。
- 窗口保持最小尺寸时，所有标题、数值、图表与选项均完整可见。

## 验收标准

- 启动详细看板时默认尺寸为 `920 × 620`。
- 看板中不存在垂直或水平滚动条。
- 外层 5 小时环与内层 7 天环使用不同颜色，且各自的剩余比例和重置时间清晰可辨。
- 今日、7 日、30 日和累计四个宫格均同时展示总量、输入、缓存、输出和推理。
- 四个宫格内的输入、缓存、输出和推理之和等于对应周期总量。
- 页面不展示总额度进度条和模型分布卡片。
- Token 趋势、Token 构成和时间范围选择器保持可见。
- 顶部周期总量与下方 Token 构成遵循一致且不重复的 Token 口径。
- 窗口无法缩小到 `920 × 620` 以下。
- 菜单栏弹窗布局和行为不受影响。
- Xcode Debug 构建和现有单元测试通过。
